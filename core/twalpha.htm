<:>theWord alpha version

Note! If you have never downloaded theWord Alpha version from Add Titles you will need at least this file to get started. Download this and replace your theword.exe with it.

Note this will only work correctly if start from "theword.exe" not a different name and you must have the file above first!


twalpha is the alpha version of theWord where the new features are first added and become available for early testing. Please only discuss twalpha in the github and not in theWord forum. The forum is for theWord beta and beyond. If you are unable to access the github please request permission at twalpha@theword.net but first make sure to read the rest of this book and setup a github account. Then when you request permission include the email used for your github account. 



<:>Alpha version Installation

  1. Testing in a new compact install is recommended (We don't recommend a normal/full installation of the alpha version.)
  2. To start off install theWord in compact installation using something like https://www.theword.net/files/setup/getmulti.php/en-small.
  3. Always keep backups of theWord.
  4. Always read the changelog-alpha.txt (For example there are some versions that can reindex your entire library!)


<:>Issues/Bugs

  1. Before submitting any issues/bugs please read the changelog thoroughly.
  2. Any issues found should be filed here and use the tag "bug" or put [BUG] in the topic.
  3. Please don't post anything regarding the alpha at theWord forums. We don't want to confuse users of the stable (beta) version.
  4. Please include the twalpha version: 6.0.1.1533.alpha.23
  5. twalpha can contain several new features at once, so it's important to mention the specific feature being tested, ideally as a tag: bible-sets
  6. Please included detailed steps to reproduce the bug and snapshots/pictures.
  7. Please include: Operating system: Windows 10, 11; Linux Version etc.


<:>New Features Requests

  1. Please no hen pecking, that is constantly asking for new features/changes. This could lead to removal from alpha testing.
  2. You may ask for features but do it throughly here and use the tag "enhancement" or put [ENHANCEMENT] in the topic.
  3. Note: Please focus on bugs instead of new features/changes.


<:>Summary of main theWord alpha features

This may not get updated frequently so make sure to check out twAlpha Changelog (or changelog-alpha.txt) for each update. These are in order of first introduction roughly. Please refer to these when addressing a bug, feature. (Programmers feel free to add to this list.)

  • Bible sets in Bible view
  • 4 Hot keys added to Book View for User Edit mode. Alt+Right was added as a hotkey for Indent Alt+Left was added as a hotkey for Decrease indent Ctrl+Alt+C was added a hot key for applying color Ctrl+Alt+H was added a hot key for applying highlight
  • Show/Hide Verse Numbers in Bible view
  • Statistics/New Search Features added to Bible search view Detailed tab: Checkbox Matches, Checkbox Statistics Options tab: NEARw (words) applies to whole clause NEARw (words) applies to original only Important: Review the popups for Nxt, TR, NearW, {} -- new features added. Some take time. Statistics tab: Shows if enabled in detailed tab. Various itmes best to try it out.
  • topic_wordindex ver6 introduced (works in the background) //Everything should be normalizedC this is a unicode standard or D but C to help theWord be one version //Hebrew can have a special unicode standard which actually breaks the standard //see here: https://www.babelstone.co.uk/Software/BabelPad_Custom_Normalizations.html#Hebrew //and see Appendix B here: https://www.sbl-site.org/Fonts/SBLHebrewUserManual1.5x.pdf
  • ANSI Bibles converted to UTF8 automatically if possible. Three reasons:
    1. There is much duplicate code to support ANSI which is pre-unicode for the most part.
    2. Some of the new Bible search functions don't work with the ANSI so as to not make more duplicate code.
    3. To update the code to a post unicode programming enviroment it is best to work on removing ANSI.
  • Verse detection ordinals in a Book view User Edit module updated to deal with ordinals like 1st, 2nd, 3rd. Like 1st Cor 1:1 (Some fixes to clipboard)
  • Compare View Highlight in Compare - Bible View can now highlight the difference of the first two Bibles in the view. Also if arrange Horizontally it gives a bit of a % and count. Settings for this are in Bible View Options > General Behavor... > Compare View Options
  • Book View Modules Font Greek/Hebrew Fix will now update to the default Greek/Hebrew font if the Build Font Greek/Hebrew is set for the module. These look like build.greek.font and build.hebrew.font in the config table of the module. The defaults of Gentium and Cardo are set for these if nothing has been set yet. These settings can only be changed when not in user mode. These should be a set and forget type of setting. It is best to set these on building the module. Default fonts in Preferences are now applied upon pressing Ok
  • Alpha version should now be downloaded through Add Titles
